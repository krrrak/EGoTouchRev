## 二、关键遗漏详解

### 2.1 【严重】三次多项式非线性补偿 — CoorMultiOrderFitCompensate

**TSACore 实现**：

```c
// 对每个 pitch 周期内的坐标施加对称三次多项式补偿
int CoorMultiOrderFitCompensate(int coor, double *coefs) {
    int remainder = coor % 0x400;       // 0~1023
    int x = (remainder < 0x201) ? (0x200 - remainder) : (remainder - 0x200);
    int comp = (int)(coefs[0] + coefs[1]*x + coefs[2]*x*x + coefs[3]*x*x*x);
    if (remainder >= 0x201) comp = -comp;
    return coor + comp;
}
```

- 系数 `coefs[4]` 来自 Flash 参数区 `DAT_1820d638`(X) / `DAT_1820d658`(Y)，工厂校准时写入
- 补偿在 `HPP3_CoordinateProcess` 中对 TX2 坐标执行（TX1 可选）
- **影响**：每个传感器 pitch 周期内最大可补偿 ±50~100 个内部单位（约 5~10% pitch），对边缘电极尤为关键

**你的代码**：`CoordinateSolver::Solve` 和 `StylusPipeline::Process` 中**完全没有**这一步。坐标从三角插值直接进入后处理链。

**建议实现**：

```cpp
// 在 CoordinateSolver::Solve 返回后、进入 CoorPostProcessor 之前调用
struct PitchCompensation {
    double c[4] = {0, 0, 0, 0};  // 从配置文件加载
};
int32_t ApplyPitchCompensation(int32_t coor, const PitchCompensation& p) {
    int rem = coor % 0x400;
    int x = (rem < 0x201) ? (0x200 - rem) : (rem - 0x200);
    double dx = static_cast<double>(x);
    int comp = static_cast<int>(p.c[0] + p.c[1]*dx + p.c[2]*dx*dx + p.c[3]*dx*dx*dx);
    if (rem >= 0x201) comp = -comp;
    return coor + comp;
}
```

### 2.3 【严重】三角插值公式差异

**TSACore 原始公式**：

```
offset = (L - R) × 0x200 / (2C - L - R)
```

这是标准的**抛物线顶点插值**。

**你的代码** (`TriangleAlgUsing3Point`):

```cpp
if (r < l) {
    int32_t minVal = r;
    if (p <= r) minVal = p - 1;
    int32_t den = p - minVal;
    result = -((l - minVal) * kCoorUnit) / den / 2;
} else {
    int32_t minVal = l;
    if (p <= l) minVal = p - 1;
    int32_t den = p - minVal;
    result = ((r - minVal) * kCoorUnit) / den / 2;
}
result += 0x200;
```

**关键差异**：
- TSACore 用 `(L-R)/(2C-L-R)` — 分母包含所有三点
- 你的代码用 `(X-min)/(peak-min)` — 分母只有两点差值
- 两者在信号对称时结果相近，但在信号不对称时差异达 **20~50 个内部坐标单位**（约 2~5% pitch）

**建议**：替换为标准抛物线公式：

```cpp
int32_t TriangleAlgUsing3Point(int16_t left, int16_t peak, int16_t right) {
    int32_t num = (int32_t)(left - right) * 0x200;
    int32_t den = 2 * (int32_t)peak - (int32_t)left - (int32_t)right;
    if (den == 0) return 0x200;
    return num / den + 0x200;
}
```

### 2.4 【中等】屏幕映射公式差异

**TSACore** `GetClipReport`：

```c

```

**你的代码** `BuildStylusPacket`：


**差异**：

3. TSACore 使用 `>> 10` 整数除法（四舍五入），你的代码用浮点 `lround`
4. X 轴翻转 `1.0f - gx/range` 在 TSACore 中可能由固件处理，需确认

### 2.7 【中等】起笔死区算法差异

**TSACore** `AftCoorProcess`：

```
算法：记录落笔点 → 死区内累积偏移 → 输出 = 当前坐标 - 累积偏移
特性：解锁后不突变（偏移平滑消失），边缘/中心不同阈值
```

**你的代码** `ApplyJitterLock`：

```cpp
// 简单锁定：死区内返回锁定值，超出阈值则解锁
if (dx > thr || dy > thr) {
    m_lockedCoor = cur;   // 直接跳到当前值
    return cur;
}
return m_lockedCoor;      // 返回锁定值
```

**差异**：
- TSACore 使用**偏移补偿法**：输出始终是当前坐标的函数，只是减去了起笔时的小偏移
- 你的代码使用**锁定法**：返回一个固定的锁定值，解锁瞬间会产生坐标跳变
- TSACore 区分边缘阈值 (`prmt[0xA58/59]`) 和中心阈值 (`prmt[0xA5A/5B]`)

## 三、已实现但存在参数/逻辑差异

### 3.1 IIR 压力滤波分母差异

| | TSACore | 你的代码 |
|---|---|---|
| 滤波公式 | `(64*cur + 192*prev) / 256` | `((128-w)*prev + w*cur + 64) >> 7` |
| 分母 N | 256 (Q8) | 128 (Q7) |
| 默认 α | 64/256 = 0.25 | 64/128 = 0.50 |

你的代码的 `m_pressureIirWeightQ7 = 64` 实际 α=0.50，而 TSACore 的 α=0.25。**你的压力平滑度只有 TSACore 的一半**，抖动会更明显。

**修正**：将分母从 128 改为 256，或将默认 `m_pressureIirWeightQ7` 从 64 改为 32。

### 3.2 24帧环形缓冲区方向差异

TSACore 使用 `memmove` 向后移动方式（index 0 = 最新帧），你的代码使用 `m_historyIdx` 指针递增方式。逻辑等价但 `CalcSpeed` 取最远点的方式与 TSACore 不同（TSACore 取逐帧累积路程）。
